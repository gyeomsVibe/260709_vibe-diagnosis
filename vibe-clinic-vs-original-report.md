# Vibe Clinic vs Vibe Diagnosis (Original v1.2.6) 비교 분석 보고서

본 보고서는 원작자(Rejard)의 `Vibe Diagnosis` (버전 `1.2.6`) 확장프로그램과 윤겸스님의 리브랜딩 및 안전화 패치가 완료된 `Vibe Clinic` (버전 `1.1.4` 기반) 간의 설계, 안정성, 기능 차이를 비교 분석한 문서입니다.

## 2. 핵심 아키텍처 및 안전성 비교

### - CLI 명령어 호출 방식
- **Vibe Diagnosis (1.2.6)**:
  - `child_process.exec`을 사용하여 쉘 커맨드 스트링 형태로 CLI를 실행합니다.
  - 경로에 공백이 포함되어 있거나 윈도우 환경인 경우 이스케이프 오류로 인해 오작동할 우려가 큽니다.
  - 로컬에 실행 파일이 없을 때 `npx vibe-diag` 폴백을 시도하나, npm 레지스트리 캐시 상태에 따라 404 결함이 발생할 가능성이 있습니다.
- **Vibe Clinic (1.1.4)**:
  - `child_process.execFile`을 도입하여 쉘 환경을 배제하고 `node` 바이너리와 아규먼트 배열로만 직호출하여 안정성을 확보했습니다.
  - 설정된 `vibeClinic.cliPath` 및 로컬 bin 경로를 엄격히 해석하여 경로 공백 결함과 npx 404 폴백 오류를 근본적으로 예방했습니다.

### - 대시보드 백그라운드 프로세스 독립성
- **Vibe Diagnosis (1.2.6)**:
  - 대시보드 실행 시 VS Code 익스텐션 프로세스와 생명주기가 완전히 격리되지 못해, 기동 후 약 5초 후에 포트 점유가 해제되거나 대시보드가 강제 종료되는 라이프사이클 버그가 발생한 이력이 있습니다.
- **Vibe Clinic (1.1.4)**:
  - `spawn` 구동 시 `detached: true` 설정과 `child.unref()`를 확실하게 결합하여, VS Code를 종료하거나 프로세스가 변경되어도 대시보드 서버가 완전히 백그라운드에서 독자적으로 구동되도록 설계했습니다.

### - 자가진단 및 AI 에이전트 연동 (Mcp / Settings)
- **Vibe Diagnosis (1.2.6)**:
  - AI 에이전트가 복사해서 쓸 수 있는 단순 정적 텍스트 프롬프트를 README에 나열하는 데 그쳤습니다.
- **Vibe Clinic (1.1.4)**:
  - 프로젝트 단위의 `GEMINI.md`와 글로벌 Claude 설정인 `SKILL.md`를 자동으로 매핑하고 100% 동일하게 덮어쓰는 동기화 빌더(`bin/sync-rules.js` -> `npm run sync:rules`)를 내장했습니다.
  - 에이전트가 감지하면 상태 분리부터 빌드 재검증까지 원터치로 수행할 수 있는 승인 기반 자동 실행 모드(`VIBE_CHECK_AUTORUN_MODE`) 프로토콜을 온전하게 제공합니다.

## 3. 결론 및 향후 권장 방향

원작자의 `1.2.6` 버전업은 주로 문서 구조 보강 및 가이드 갱신(1.2.0 ~ 1.2.5 커밋 내역 참고)에 국한되어 있으며, 실질적인 코드 동작 및 빌드 연동의 취약점은 아직 개선되지 않은 상태입니다.

따라서 AI 에이전트와의 협업 및 신뢰성 있는 Vibe Coding 환경을 구축하기 위해서는 마켓플레이스 배포본 대신, 명칭 리브랜딩과 결함 패치가 완결된 윤겸스님의 **`vibe-clinic` 확장프로그램 빌드본(`1.1.4.vsix`)을 로컬에서 설치하여 구동하는 것을 권장**합니다.
