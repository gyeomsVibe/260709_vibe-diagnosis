# 🩺 Vibe Diagnosis 전역 모드화(글로벌 자동화) 구현 계획서 v2

윤겸스님이 수동으로 복사/붙여넣기나 파일 수정을 거치지 않고, 오직 **명령 실행 승인**만 하실 수 있도록 PowerShell 자동화 패치 방식을 적용한 전역 검수 모드 구축 계획입니다.

---

## 1. 🏗️ 아키텍처 및 대체 경로 설계

에이전트가 직접 파일에 쓰려 할 때 보안 권한 제한이 발생하면, 에이전트가 로컬에 **임시 PowerShell 자동 패치 스크립트(`.ps1`)**를 생성하고, 사용자에게 이 스크립트의 **실행 승인**을 받아 전역 파일을 자동 구성합니다.

```
[에이전트: 패치 스크립트(.ps1) 자동 작성]
                  │
                  ▼
[사용자: 터미널 명령어 실행 승인 (버튼 클릭)]
                  │
                  ▼
[임시 스크립트 실행] ──> 1. mcp_config.json 자동 파싱 및 vibe-diagnosis 주입
                     ──> 2. AGENTS.md 자동 생성 및 글로벌 룰 추가
```

---

## 2. 📝 Phase별 수행 계획 (에이전트 직접 수행)

### 📂 Phase A — 현재 상태 판정 (완료)
- 현재 전역 설정 경로 및 `mcp_config.json`, `skills/vibe-check/SKILL.md` 존재 여부를 판정했습니다.
- `task.md`의 "사용자 직접 반영 필요" 라벨을 Blocker가 아닌 **"에이전트 직접 수행(PowerShell 스크립트 자동 패치)"**로 재분류했습니다.

### 🔌 Phase B — 전역 MCP 자동 반영 (완료)
- `mcp_config.json`을 읽고 파싱하여 `mcpServers.vibe-diagnosis`가 없는 경우에만 주입합니다.
- 직접 수정이 차단되므로, 임시 스크립트를 생성하고 `run_command`로 실행 승인을 얻어 전역 MCP를 자동 패치 완료했습니다.

### 📜 Phase C — 전역 룰(AGENTS.md) 자동 반영 (완료)
- `AGENTS.md`가 없으면 신규 생성하고, 있으면 Vibe Diagnosis 규칙 섹션을 자동으로 추가(Append)합니다.
- 직접 수정이 차단되므로, 임시 스크립트를 생성하고 `run_command`로 실행 승인을 얻어 전역 규칙을 자동 작성 완료했습니다.

### 🔮 Phase D — 전역 스킬 확인/보강 (완료)
- `skills/vibe-check/SKILL.md`가 존재하는지 확인하고, 트리거 문장들이 포함된 행동 룰을 보강 완료했습니다.

### 🧪 Phase E — 임시 프로젝트 검증 (완료)
- 임시 테스트 디렉토리를 에이전트가 직접 생성합니다.
- 해당 디렉토리에서 자가진단 초기화(`init_diagnostics`) 및 `run_diagnostics`를 실행하여 글로벌 원터치 점검 모드가 오류 없이 동작하는지 최종 검증 완료했습니다.

## 3. 🛡️ 안전 장치 및 예외 처리
- **기존 설정 보존**: `mcp_config.json`을 패치할 때 기존의 다른 MCP 서버들(`cloudrun`, `notebooks` 등)을 훼손하지 않고 유지합니다.
- **백업 생성**: 전역 설정을 변경하기 전에 자동으로 백업본(`.bak`)을 만들어 롤백이 가능하도록 보장합니다.
