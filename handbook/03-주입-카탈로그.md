# 03. 주입 카탈로그 — 무엇을 망가뜨려 볼까

> 앞: [02. 실전 5단계](02-실전-5단계.md) · 다음: [04. AI에게 시키는 법](04-AI에게-시키는-법.md)

## 이 문서를 쓰는 법

매번 "뭘 테스트하지?" 하고 고민하지 마세요. **위에서부터 순서대로 내려가면 됩니다.**

퍼징 연구가 알려주는 교훈이 이겁니다:
**당신이 떠올릴 수 있는 문제는 이미 처리했을 가능성이 높습니다. 위험한 건 떠올리지 못한 쪽입니다.**
그래서 목록이 필요합니다.

- ⭐⭐⭐ = 가장 자주 결함이 나옴. **여기부터 하세요**
- ⭐⭐ = 자주 나옴
- ⭐ = 상황에 따라

---

## A. 양(量) — 개수와 길이

### A1. 텅 빈 것 ⭐⭐⭐

**주입**: 목록 0개, 검색 결과 없음, 빈 문자열, 값 없음(null)

**왜 위험한가**: 만들 때는 항상 데이터가 있는 상태로 봅니다. 0개인 화면은 아무도 안 봅니다.

**확인 포인트**
- [ ] 빈 화면에 **왜 비었는지 설명**이 있나? (그냥 흰 화면 아님)
- [ ] "첫 항목을 추가하세요" 같은 **다음 행동**을 알려주나?
- [ ] 개수 표시가 `0개`인가, 아니면 `undefined개`인가?
- [ ] 평균·비율 계산이 있다면 → **0으로 나누기**가 터지지 않나?

### A2. 딱 하나 ⭐⭐

**주입**: 항목 1개만

**왜 위험한가**: "3개 중 2번째" 같은 표시, 목록 사이 구분선, 좌우 이동 버튼이 1개일 때 이상해집니다.

**확인 포인트**
- [ ] 구분선이 하나 남아 떠 있지 않나?
- [ ] "이전/다음" 버튼이 눌리면 안 되는데 눌리나?
- [ ] `1개의 항목들` 같은 어색한 복수형이 없나?

### A3. 너무 많은 것 ⭐⭐

**주입**: 1,000개, 10,000개

**확인 포인트**
- [ ] 화면이 멈추거나 심하게 느려지지 않나?
- [ ] 스크롤이 되나?
- [ ] 개수 표시가 `1000+` 처럼 정리되나, 아니면 그대로 다 나오나?

### A4. 아주 긴 글자 ⭐⭐⭐

**주입**: 제목에 500자, 5,000자

**왜 위험한가**: 이름은 짧을 거라고 **무의식적으로 가정**합니다. 레이아웃이 가장 잘 깨지는 지점입니다.

**확인 포인트**
- [ ] 글자가 상자를 뚫고 나가지 않나?
- [ ] 가로 스크롤이 생기지 않나?
- [ ] 옆 요소를 밀어내지 않나?
- [ ] 말줄임(`…`) 처리가 되나?

### A5. 띄어쓰기 없는 긴 문자열 ⭐⭐

**주입**: `AAAAAAAA...`를 200자 (공백 없이)

**왜 위험한가**: 줄바꿈은 보통 띄어쓰기에서 일어납니다. 공백이 없으면 **한 줄이 무한정 길어집니다.**
긴 URL이나 파일 경로에서 실제로 자주 발생합니다.

## B. 글자 — 내용의 모양

### B1. 특수문자와 태그 ⭐⭐⭐

**주입**: `<b>굵게</b>`, `<script>alert(1)</script>`, `"큰따옴표"`, `'작은따옴표'`, `&앰퍼샌드`

**왜 위험한가**: 두 가지 문제가 있습니다.
1. 태그가 **글자로 안 보이고 진짜 태그로 동작** → 보안 구멍(XSS)
2. 따옴표가 문장을 깨뜨림

**확인 포인트**
- [ ] `<b>굵게</b>`가 **글자 그대로** 보이나? (굵게 변하면 위험)
- [ ] 스크립트가 실행되지 않나?
- [ ] 따옴표가 들어간 이름이 깨지지 않나?

### B2. 이모지와 다국어 ⭐⭐

**주입**: `🩺💉🔥`, `한글`, `日本語`, `العربية`(오른쪽에서 왼쪽 씀)

**확인 포인트**
- [ ] 이모지가 `?` 나 네모(□)로 안 나오나?
- [ ] 글자 수 세기가 이상하지 않나? (이모지 1개가 2~4자로 세지기도 합니다)
- [ ] 아랍어 같은 우→좌 문자에서 방향이 뒤집히지 않나?

### B3. 앞뒤 공백과 줄바꿈 ⭐

**주입**: `"  이름  "`, 줄바꿈이 들어간 제목

**확인 포인트**
- [ ] 공백이 정리되나?
- [ ] 한 줄짜리 칸에 여러 줄이 들어와 깨지지 않나?

---

## C. 시간 — 빠르기

### C1. 아주 느림 ⭐⭐⭐

**주입**: 응답을 10초, 30초 지연

**왜 위험한가**: 만들 때는 내 컴퓨터에서 **즉시** 응답합니다. 로딩 상태를 볼 일이 없습니다.

**확인 포인트**
- [ ] 로딩 표시가 **나오나?**
- [ ] 기다리는 동안 버튼을 **또 누를 수 있나?** (막아야 합니다)
- [ ] 아주 오래 걸리면 **포기하고 안내**하나, 아니면 영원히 도나?

### C2. 순서 뒤바뀜 ⭐

**주입**: 빠르게 검색어를 바꿔서 이전 요청이 나중에 도착하게 만들기

**확인 포인트**
- [ ] "사과" 검색 후 "바나나" 검색했는데 **사과 결과**가 나오지 않나?

---

## D. 연결 — 통신

### D1. 끊김 ⭐⭐⭐

**주입**: 인터넷 끊기 (비행기 모드, 네트워크 차단)

**확인 포인트**
- [ ] "인터넷 연결을 확인하세요" 같은 **사람 말**이 나오나?
- [ ] `Failed to fetch` 같은 **영어 원문**이 그대로 나오지 않나?
- [ ] 다시 연결되면 **회복**되나? 아니면 새로고침해야 하나?

### D2. 서버 오류 ⭐⭐⭐

**주입**: 500(서버 오류), 404(없음), 403(권한 없음), 429(너무 많은 요청)

**왜 위험한가**: 각각 **사용자가 할 수 있는 일이 다릅니다.** 전부 "오류 발생"으로 뭉뚱그리면 안 됩니다.

| 코드 | 사용자가 할 일 | 안내 문구 예시 |
|---|---|---|
| 500 | 기다렸다 다시 | "일시적인 문제입니다. 잠시 후 다시 시도해 주세요" |
| 404 | 다른 걸 찾기 | "찾는 항목이 없습니다" |
| 403 | 로그인/권한 요청 | "권한이 필요합니다" |
| 429 | 잠시 쉬기 | "요청이 많습니다. 1분 후 다시 시도해 주세요" |

### D3. 이상한 응답 ⭐⭐

**주입**: 빈 응답, 잘린 JSON, HTML이 와야 할 곳에 JSON

**확인 포인트**
- [ ] 화면이 하얗게 되지 않나?
- [ ] `[object Object]` 나 `undefined` 가 화면에 보이지 않나?

## E. 형태 — 데이터 모양 ⭐⭐⭐ (가장 많이 걸림)

> **이 항목이 우리 프로젝트에서 결함 3건 중 3건 전부의 원인이었습니다.**

### E1. 예상과 다른 형태

**주입**
- 글자인 줄 알았는데 **목록**이 옴 (`"조치"` ↔ `["조치1","조치2"]`)
- 목록인 줄 알았는데 **하나**만 옴
- 글자인 줄 알았는데 **덩어리(객체)** 가 옴

**왜 위험한가**: 화면에 덩어리를 그대로 그리려 하면 **화면 전체가 터집니다.**
`[object Object]`가 보이면 이 문제입니다.

**확인 포인트**
- [ ] 두 형태 다 넣어보고 화면이 살아 있나?
- [ ] `[object Object]` 가 보이지 않나?

### E2. 값이 없음 ⭐⭐⭐

**주입**: `null`, `undefined`, 필드 자체가 없음

**확인 포인트**
- [ ] `undefined` 나 `null` 이라는 **글자가 화면에 보이지 않나?**
- [ ] 날짜가 `Invalid Date` 로 나오지 않나?
- [ ] 숫자가 `NaN` 으로 나오지 않나?

### E3. 이름이 다른 필드 ⭐⭐⭐

**주입**: 서버가 주는 이름과 화면이 찾는 이름을 어긋나게 만들기

**왜 위험한가**: **가장 조용한 결함**입니다. 오류도 안 나고, 그냥 **아무것도 안 보입니다.**

우리가 실제로 겪은 사례:
> 화면은 `results` 라는 이름을 찾는데, 서버는 `cured`, `manual` 이라는 이름으로 보냅니다.
> 찾는 이름이 없으니 **"없으면 빈 목록"** 이라는 대비책이 작동합니다.
> 오류 없이, 화면은 **영원히 비어 있습니다.**

**확인 포인트**
- [ ] 항상 비어 있는 영역이 있다면 → **이름이 틀렸을 가능성이 큽니다**
- [ ] "없으면 빈 값" 처리가 **정상 경로를 가리고 있지 않나?**

### E4. 표시용 이름을 식별자로 쓰기 ⭐⭐

**왜 위험한가**: 이것도 우리가 실제로 겪었습니다.
> 목록에서 "Google Gemini"를 고르면, 화면은 **보이는 글자**를 저장했습니다.
> 시스템은 `gemini` 라는 **내부 이름**으로 찾습니다. 저장은 "성공"이라고 떴고,
> **그 뒤 모든 AI 기능이 죽었습니다.**

**확인 포인트**
- [ ] 선택 목록에서 **보이는 글자**와 **저장되는 값**이 분리돼 있나?
- [ ] 저장 후 그 값을 **실제로 쓰는 기능까지** 한 번 돌려봤나?

## F. 권한과 상태

### F1. 로그아웃/만료 ⭐⭐

**주입**: 로그인 상태에서 세션 만료시키기

**확인 포인트**
- [ ] 로그인 화면으로 **안내**하나, 아니면 그냥 오류인가?
- [ ] 쓰던 내용이 **날아가나?**

### F2. 권한 없는 접근 ⭐⭐

**주입**: 남의 항목 열어보기, 없는 주소 직접 입력

**확인 포인트**
- [ ] 막히나?
- [ ] 오류 메시지에 **없어야 할 정보**가 새지 않나? (내부 경로, 다른 사람 이름 등)

## G. 순서와 타이밍

### G1. 버튼 연타 ⭐⭐⭐

**주입**: 저장 버튼을 아주 빠르게 5번 클릭

**확인 포인트**
- [ ] **5개가 저장되지 않나?**
- [ ] 첫 클릭 후 버튼이 **잠기나?**

### G2. 중간에 이탈 ⭐⭐

**주입**: 처리 중에 뒤로가기, 새로고침, 창 닫기

**확인 포인트**
- [ ] 반쯤 저장된 이상한 상태가 남지 않나?
- [ ] "저장 중입니다" 경고가 뜨나?

### G3. 두 곳에서 동시에 ⭐

**주입**: 창 두 개를 열고 같은 걸 동시에 수정

**확인 포인트**
- [ ] 나중 것이 앞의 것을 **조용히 덮어쓰지** 않나?

## 빠른 실행표 (처음 30분)

시간이 없다면 이 여섯 개만 하세요. **결함 발견율이 가장 높습니다.**

| 순서 | 주입 | 확인 |
|---|---|---|
| 1 | **목록 0개** | 안내 문구가 있나? |
| 2 | **긴 이름(500자)** | 레이아웃이 버티나? |
| 3 | **`<b>태그</b>` 입력** | 글자로 보이나? |
| 4 | **인터넷 끊기** | 사람이 읽을 메시지가 나오나? |
| 5 | **서버 오류(500)** | 다음에 뭘 할지 알려주나? |
| 6 | **저장 버튼 연타** | 중복 저장이 안 되나? |

## 나만의 카탈로그 만들기

이 목록은 **출발점**입니다. 만드는 것마다 위험한 지점이 다릅니다.

물어볼 질문 세 개:

1. **"이 화면에, 문제가 생겼을 때만 나타나는 부분이 있나?"**
   → 있다면 1순위입니다. ([01. 백신 원리 §7](01-백신-원리.md) 참고)
2. **"내가 '이건 항상 이럴 거야'라고 가정한 게 뭐지?"**
   → 그 가정을 깨보세요.
3. **"실제 사용자가 가장 자주 겪을 실패는?"**
   → 공식 원칙도 **"현실에서 일어날 법한 일을 우선"** 하라고 말합니다.

새로 발견한 주입은 이 문서에 **추가하세요.** 카탈로그는 자라야 합니다.
