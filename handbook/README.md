# 백신 개발법 (The Vaccine Method)

> **일부러 아프게 해서, 진짜 아플 때를 대비한다.**
>
> 만든 것이 조용히 고장 나 있는 걸 미리 찾아내는 방법. 코딩을 몰라도 쓸 수 있습니다.

---

## 이 교본이 다루는 문제

무언가를 만들었습니다. 열어보니 잘 됩니다. 검사도 다 통과했습니다.
**그런데 실제로 쓰는 사람은 얼마 못 가 "안 돼요"라고 합니다.**

왜 이런 일이 생길까요?

당신이 확인한 건 **잘 될 때의 모습** 하나뿐이기 때문입니다.
소프트웨어에는 얼굴이 두 개 있습니다. 잘 될 때의 얼굴, 그리고 **문제가 생겼을 때의 얼굴.**
두 번째 얼굴은 평소에 절대 나타나지 않습니다. 그래서 아무도 안 봅니다.
결함은 바로 거기 숨어 삽니다.

이 교본은 **그 두 번째 얼굴을 일부러 불러내는 방법**을 알려줍니다.

---

## 왜 하필 "백신"인가

예방주사를 생각해 보세요. 병을 막으려고 **일부러 약한 병원체를 몸에 넣습니다.**
몸은 그걸 겪어보고 항체를 만듭니다. 그래서 진짜 병이 왔을 때 이깁니다.

한 번도 아파본 적 없는 몸은 강한 게 아니라, **아직 시험받지 않은 것**입니다.

소프트웨어도 똑같습니다.

| 몸 | 소프트웨어 |
|---|---|
| 약한 병원체를 일부러 주입 | 통제된 실패를 일부러 만듦 |
| 격리된 환경에서 안전하게 | 진짜 데이터 말고 복제본에서 |
| 면역 반응 관찰 | 화면이 어떻게 되는지 관찰 |
| 항체가 남음 | 재발 방지 검사가 남음 |
| 다음엔 안 걸림 | 같은 결함이 다시 안 생김 |

이 비유는 그냥 말장난이 아닙니다. 실제 이론적 뿌리가 같습니다 —
자세한 내용은 [01. 백신 원리](01-백신-원리.md)에서 다룹니다.

## 5분 요약

### 핵심 문장 세 개

1. **잘 되는 것만 확인한 건, 절반만 확인한 것이다.**
2. **문제가 생겼을 때만 나타나는 화면이 있다. 거기가 결함의 서식지다.**
3. **검사를 다 통과해도 결함은 살아 있을 수 있다.** 검사는 당신이 상상한 것만 확인한다.

### 절차 다섯 단계

```
① 격리     진짜 말고 복제본에서 한다          "환자 말고 배양접시에서"
② 기록     멀쩡할 때 모습을 먼저 적어둔다      "평소 체온을 알아야 열을 안다"
③ 주입     실패를 일부러 하나 만든다           "약한 병원체를 넣는다"
④ 관찰     화면을 직접 열어서 눈으로 본다      "증상을 본다"
⑤ 항체     고치고, 재발 방지 검사를 남긴다      "면역을 남긴다"
```

한 번에 **한 가지 실패만** 주입하세요. 두 개를 동시에 넣으면 뭐가 원인인지 모릅니다.

### 무엇을 주입할까 (가장 잘 걸리는 8가지)

| 주입 | 한 줄 설명 |
|---|---|
| **텅 빈 것** | 목록이 0개, 이름이 빈칸, 값이 없음 |
| **너무 많은 것** | 1000개, 5000글자짜리 제목 |
| **이상한 글자** | 이모지, `<태그>`, 따옴표, 아주 긴 한글 |
| **느린 것** | 응답이 30초 걸림 |
| **끊긴 것** | 인터넷 끊김, 서버 오류 |
| **형태가 다른 것** | 글자인 줄 알았는데 목록이었음 |
| **권한 없음** | 로그아웃 상태, 접근 거부 |
| **두 번 누르기** | 버튼 연타, 뒤로가기 |

전체 목록은 [03. 주입 카탈로그](03-주입-카탈로그.md)에 있습니다.

### AI에게 시킬 때 쓰는 한 문장

> "정상 상태 말고, **실패한 상태에서** 같은 화면을 다시 열어봐.
> 실패했을 때만 생기는 데이터가 화면을 깨뜨리는지 확인해줘."

더 좋은 프롬프트는 [04. AI에게 시키는 법](04-AI에게-시키는-법.md)에 있습니다.

## 이 방법이 실제로 잡은 것

이 교본은 책상에서 지어낸 게 아닙니다.
Vibe Clinic 프로젝트에서 **검사 47개를 전부 통과하고 코드 검사도 깨끗한 상태**로
이 방법을 적용했더니, 하루에 결함 **3건**이 나왔습니다.

- 실패한 항목을 누르면 **화면 전체가 하얗게 사라짐**
- 설정을 저장하면 **AI 기능이 통째로 죽음** (저장은 "성공"이라고 표시됨)
- 결과 보고서가 **숫자만 보여주고 내용은 항상 비어 있음**

셋 다 **정상 상태에서는 절대 재현되지 않는** 것이었습니다.
자세한 경위는 [05. 실증 사례](05-실증-사례.md)에 있습니다.

## 한 가지 경고

이 방법의 목적은 **부수는 게 아니라 배우는 것**입니다.

- 진짜 데이터, 실제 서비스, 남이 쓰고 있는 것에 하지 마세요. **반드시 복제본에서.**
- 실패를 주입했으면 **반드시 원래대로 되돌리세요.**
- 안 되는 걸 확인하는 데서 끝내지 말고, **고치고 항체를 남기세요.**

이 세 가지를 지키면 안전합니다.
